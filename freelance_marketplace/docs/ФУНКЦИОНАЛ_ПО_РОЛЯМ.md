# Функционал пользователей Freelance Marketplace

Документ описывает возможности платформы в зависимости от роли. В системе четыре базовые роли в БД: **executer** (исполнитель), **customer** (заказчик), **manager** (менеджер), **administrator** (администратор). Пользователь может иметь **несколько ролей одновременно** — интерфейс показывает объединённый набор функций.

Тестовые учётки из `data.sql` (пароль для всех: `Qwert123_`):

| Логин | Роли | Назначение |
|--------|------|------------|
| `customer` | заказчик | Только заказчик |
| `executer` | исполнитель | Только исполнитель |
| `ssjjitt` | заказчик + исполнитель | Обе роли |
| `manager` | менеджер | Поддержка и модерация |
| `admin` | администратор | Управление платформой |
| `antikvar`, `savko` | заказчик / исполнитель | Доп. демо-данные |

## 1. Гость (не авторизован)

Доступ без входа (`ProtectedRoute` с `allowPublic`).

### Доступные страницы

| Раздел | Маршрут | Что можно |
|--------|---------|-----------|
| Главная | `/#/` | Просмотр лендинга, поиск, категории, блоки «популярное» (без персональных кнопок заказчика/исполнителя) |
| Каталог | `/#/catalog` | Просмотр заказов, услуг, пользователей; фильтры и сортировка |
| Карточка заказа | `/#/orders/:id` | Просмотр деталей заказа (без отклика) |
| Карточка услуги | `/#/services/:id` | Просмотр услуги (без отклика) |
| Профиль пользователя | `/#/profile/:id` | Просмотр публичного профиля |
| Вход | `/#/login` | Авторизация |
| Регистрация | `/#/register` | Регистрация в 3 шага |
| Подтверждение email | `/#/verify` | Ввод кода из письма |

### Ограничения

- Нет чатов, уведомлений, избранного, создания заказов/услуг, откликов, личного кабинета.
- При попытке открыть защищённый раздел показывается форма «Требуется авторизация» (вход / переход на регистрацию).

### Регистрация

1. Логин, email, пароль, подтверждение пароля.
2. Выбор ролей: **заказчик** и/или **исполнитель**; подтверждение email кодом.
3. Завершение регистрации.

## 2. Заказчик (customer)

Роль для размещения заказов и поиска исполнителей.

### Главная страница (`/#/`)

- Поиск по заказам, услугам и пользователям.
- Блок **«Разместить заказ»** → создание заказа.
- Популярные и новые категории (переход в каталог с фильтром).
- Лента недавних **услуг** исполнителей.
- Рейтинг лучших фрилансеров и заказчиков (просмотр).

### Заказы

| Действие | Где | Описание |
|----------|-----|----------|
| Создать заказ | `/#/orders/new`, меню «Новый заказ» | Название, описание, бюджет, срок, категория, вложения |
| Редактировать свой заказ | `/#/orders/:id/edit` | Только свои заказы |
| Удалить свой заказ | Карточка заказа | Только владелец |
| Завершить заказ | Карточка заказа | Статусы `open` / `in_progress` → `completed` |
| Просмотр откликов | Карточка своего заказа | Список откликов исполнителей |
| Одобрить отклик | Карточка заказа | Выбор исполнителя (статус отклика `approved`) |
| Отклонить отклик | Карточка заказа | Отклонение других откликов |
| Оставить отзыв | После `completed` | Оценка исполнителю по завершённому заказу |
| Пожаловаться на заказ | Чужой заказ | Жалоба менеджеру |

Статусы заказа: открыт, в работе, завершён, отменён, спор, скрыт, закрыт. Бейдж **«Одобрено менеджером»** — доверие модерации.

### Услуги (как заказчик)

| Действие | Где | Описание |
|----------|-----|----------|
| Просмотр каталога услуг | `/#/catalog` | Вкладка «Услуги» |
| Откликнуться на услугу | `/#/services/:id` | Сообщение и предложенная цена |
| Отменить свой отклик | Карточка услуги | Пока отклик в статусе «на рассмотрении» |
| Оставить отзыв | После завершения | Исполнителю по сделке |

### Каталог (`/#/catalog`)

- Вкладки: все / заказы / услуги / пользователи.
- Фильтр по **категориям** (дерево в боковой панели и в меню).
- Фильтр **цены** (от–до).
- Фильтр **статуса заказов** (только для заказчиков).
- **Поиск** и **сортировка** (дата, цена, название и др.).
- Переход в карточки заказов, услуг, профилей.

### Избранное (`/#/favorites`)

- Добавление исполнителей в избранное с их публичного профиля (кнопка «В избранное»).
- Список избранных, удаление из списка.

### Отклики (`/#/applications`, `/#/applications/history`)

- **Отклики к моим объявлениям** — входящие отклики на свои заказы и услуги; одобрение/отклонение.
- **История моих откликов** — отклики, которые заказчик сам отправил на чужие заказы/услуги.
- Открытие **спора** по отклонённому отклику (если доступно в UI).

### Профиль

| Раздел | Маршрут | Возможности |
|--------|---------|-------------|
| Свой профиль | `/#/profile` | Просмотр статистики, отзывов, навыков |
| Редактирование | `/#/profile/edit` | ФИО, город, о себе, навыки, контакты, аватар |
| Чужой профиль | `/#/profile/:id` | Просмотр; для исполнителей — избранное, чат, отзыв, жалоба |

### Коммуникация

| Раздел | Маршрут | Возможности |
|--------|---------|-------------|
| Чаты | `/#/chats` | Переписка с исполнителями; создание чата из профиля |
| Уведомления | `/#/notifications` | Входящие события: отклики, статусы, сообщения, оценки; фильтр «Все» / «Непрочитанные»; отметка прочитанным |

### Управление контентом

| Раздел | Маршрут | Возможности |
|--------|---------|-------------|
| Мои заказы/услуги | `/#/my-items` | Вкладка «Мои заказы»: список, редактирование, удаление, переход в карточку |

*(Услуги как исполнитель на этой странице недоступны без роли executer.)*

### Общее для авторизованных

- Поиск в шапке сайта.
- Боковое меню: главная, чаты, уведомления, каталог, категории, профиль, настройки, выход.
- Переключение светлой/тёмной темы.
- При блокировке аккаунта — экран «Аккаунт заблокирован» с выходом.

## 3. Исполнитель (executer)

Роль для публикации услуг, откликов на заказы и ведения резюме.

### Главная страница (`/#/`)

- Поиск.
- Блок **«Разместить услугу»** → `/#/services/new`.
- Категории, недавние **заказы**, рейтинги.

### Услуги

| Действие | Где | Описание |
|----------|-----|----------|
| Создать услугу | `/#/services/new`, меню «Новая услуга» | Название, описание, цена, категория, вложения |
| Редактировать | `/#/services/:id/edit` | Свои услуги |
| Удалить | Карточка услуги | Своя услуга |
| Статус active/inactive | Управление | Публикация и снятие с показа |
| Просмотр откликов заказчиков | Своя услуга | Одобрение/отклонение откликов на услугу |

После создания услуга может требовать **одобрения менеджером** (`isApproved`).

### Заказы (как исполнитель)

| Действие | Где | Описание |
|----------|-----|----------|
| Откликнуться | `/#/orders/:id` | На открытые заказы: сообщение, цена |
| Отменить отклик | Карточка заказа | Пока статус «на рассмотрении» |
| Спор | Карточка заказа | При отклонённом отклике — открытие спора |
| Оставить отзыв | После `completed` | Заказчику, если отклик был одобрен |

### Резюме

| Действие | Где | Описание |
|----------|-----|----------|
| Создать резюме | `/#/resumes/new` | Опыт, образование, навыки, портфолио |
| Редактировать | `/#/resumes/:id/edit` | Свои резюме |
| Просмотр | `/#/resumes/:id` | Публичный просмотр после одобрения |

Резюме проходит **модерацию** (менеджер или администратор на главной / в админ-разделах).

### Каталог, отклики, профиль, чаты

- Те же возможности каталога, что у заказчика (без фильтра статусов заказов, если нет роли customer).
- **История откликов** и **отклики к моим услугам** — аналогично заказчику, но с фокусом на услуги.
- **Мои заказы/услуги** (`/#/my-items`) — вкладка **«Мои услуги»**.

### Недоступно без роли customer

- Создание заказов, избранное исполнителей, блок «Разместить заказ» на главной.

## 4. Менеджер (manager)

При входе менеджер **перенаправляется на** `/#/manager` (не на обычную главную).

### Панель менеджера (`/#/manager`)

- Сводка: открытые **тикеты**, **споры**, активные **пользователи**, модерация **заказов**.
- Быстрые ссылки на все подразделы модерации.

### Тикеты поддержки

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/tickets` | Список обращений пользователей; фильтр по статусу |
| `/#/manager/tickets/:id` | Просмотр тикета, смена статуса, назначение менеджера |

Категории тикетов: техническая, платежи, спор, нарушение, прочее. Приоритеты: низкий — срочный.

### Споры

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/disputes` | Список споров по заказам/сделкам |
| `/#/manager/disputes/:id` | Рассмотрение, **разрешение спора** (в т.ч. компромисс) |

### Модерация пользователей

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/users` | Поиск пользователя, просмотр карточки |
| — | **Блокировка** / **разблокировка** с указанием причины |

### Модерация заказов

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/orders` | Список заказов платформы |
| — | Скрыть / показать, закрыть / открыть заказ |
| — | Выдача бейджа **«Одобрено менеджером»** (доверие) |

### Модерация услуг

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/services` | Список услуг |
| — | Скрытие, закрытие, возврат в каталог |

### Модерация резюме

| Маршрут | Функционал |
|---------|------------|
| `/#/manager/resumes` | Очередь резюме на проверку |
| — | Одобрение / отклонение |

### Общий доступ менеджера

- Каталог, профили, чаты, уведомления — как у обычного пользователя (для коммуникации).
- API также разрешает те же действия, что и администратору, в рамках модерации (блокировка, отчёты).

### Недоступно

- Создание заказов/услуг от имени менеджера через стандартные формы заказчика/исполнителя (нет этих ролей в UI по умолчанию).
- Админ-панель на главной с массовой рассылкой и полной статистикой (это зона администратора, хотя часть модерации дублируется).

## 5. Администратор (administrator)

Полный доступ к управлению платформой. На главной (`/#/`) вместо лендинга заказчика показывается **админ-дашборд**.

### Статистика и аналитика (главная)

- Счётчики: пользователи, заказы, услуги, активные заказы.
- **Статистика за месяц** (календарь): заказы, услуги, выполненные заказы, регистрации.
- Лента активности платформы.

### Управление категориями (главная и `/admin`)

- Создание **категории** и **подкатегории** (название, описание, родитель).
- Просмотр списка категорий на `/#/admin`.

### Модерация резюме

- На главной: очередь резюме → **одобрить**.
- `/#/admin/resumes` — список резюме, одобрение/отклонение.
- `/#/admin/moderation` — сводная статистика модерации (резюме и услуги).

### Рассылка уведомлений

- На главной: форма **массового уведомления** всем пользователям (заголовок + текст).

### Админ-панель (`/#/admin`)

| Функционал | Описание |
|------------|----------|
| Список пользователей | Все пользователи, роли |
| Блокировка / разблокировка | Как у менеджера |
| Управление категориями | Создание, удаление |
| Резюме на модерации | Одобрение из панели |

### Модерация услуг

- `/#/admin/services` — просмотр и модерация услуг (аналог менеджерского раздела).

### Всё, что доступно менеджеру

Администратор может открыть разделы `/#/manager/*` (в маршрутах указаны роли `manager` и `administrator`):

- тикеты, споры, пользователи, заказы, услуги, резюме.

### Дополнительно на чужих профилях

- Кнопки **заблокировать / разблокировать** пользователя (как на публичном профиле).

### Общий функционал

- Чаты, уведомления, каталог, поиск — для контроля и поддержки.
- В боковом меню для admin: пункт «Модерация резюме» → `/admin/resumes` (без пунктов «Мои заказы» / «Отклики», если не dual-role).

## 6. Пользователь с несколькими ролями

Пример: **`ssjjitt`** — заказчик + исполнитель.

### Интерфейс

- На главной отображаются **оба** блока: «Разместить заказ» и «Разместить услугу».
- В меню: «Новый заказ» и «Новая услуга».
- **Мои заказы/услуги** — обе вкладки.
- **Избранное** — доступно как заказчик.
- Отклики: и на чужие заказы (исполнитель), и входящие на свои объявления (заказчик/исполнитель).

Логика UI строится по проверке `hasCustomerRole` / `hasExecuterRole` — каждая возможность включается независимо.

Пользователи получают уведомления в разделе `/#/notifications`:

## 9. Технические примечания

- Маршрутизация: **HashRouter** (`/#/path`). При прямом переходе на `/login` без hash может потребоваться `/#/login`.
- Роли в JWT/localStorage: `ROLE_CUSTOMER`, `ROLE_EXECUTER` и т.д.; проверка на фронте по подстроке в названии роли.
- Заблокированный пользователь (`isBlocked`): при входе показывается экран блокировки, доступ к API ограничен.
- Публичные маршруты: главная, каталог, карточки заказа/услуги/профиля по id — без обязательного входа.
